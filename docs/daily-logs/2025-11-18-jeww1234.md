- ## 상세

  ### 로그인

  - 내용: React 프로젝트에서 Google OAuth 로그인을 구현하기 위해 @react-oauth/google 라이브러리를 사용하고, JWT 토큰을 디코딩하여 사용자 정보를 상태 관리 라이브러리 Zustand에 저장하는 방식으로 구성했습니다

  사용한 라이브러리

  - @react-oauth/google : 구글 로그인 컴포넌트 제공
  - jwt-decode : JWT 토큰 디코딩
  - zustand : 전역 상태 관리

  환경 변수 설정 : .env 파일에 Google OAuth Client ID를 저장합니다.
  VITE*GOOGLE_CLIENT_ID=YOUR_GOOGLE_CLIENT_ID(vite 환경이라면 VITE* 는 필수)

  LoginPage 컴포넌트 : 구글 로그인 버튼을 표시하고, 로그인 성공 시 JWT를 디코딩하여 사용자 정보를 저장합니다.

  main.jsx 설정 : Google OAuth Provider로 앱 전체를 감싸서 로그인 기능을 사용할 수 있도록 합니다.

  ### 제이슨 서버

  - 내용: React 프로젝트에서 JSON Server를 생성하고, fetch를 통해 데이터를 가져온 뒤 로그인한 사용자 이름과 비교하여 해당 사용자와 일치하는 항목만 화면에 표시하도록 구성했습니다.

  사용한 라이브러리

  - json-server : 로컬 API 서버 생성
  - zustand : 로그인 사용자 상태 관리
  - jwt-decode : JWT 토큰에서 사용자 정보 추출

  환경 변수 설정 : .env 파일에 Google OAuth Client ID를 저장합니다.
  VITE*GOOGLE_CLIENT_ID=YOUR_GOOGLE_CLIENT_ID(vite 환경이라면 VITE* 는 필수)

  DiaryListPage 컴포넌트 : 로그인 성공 시 발급되는 JWT 토큰을 jwt-decode 라이브러리를 사용해 디코딩하면 토큰 안에 들어 있는 사용자 정보(예: name)를 추출할 수 있습니다.

  이때 추출한 name 값을 전역 상태 관리 라이브러리인 Zustand에 저장해 두면, 애플리케이션 어디서든 로그인한 사용자의 이름을 참조할 수 있습니다.

  그 다음 JSON Server에서 사용자 데이터를 가져옵니다. fetch를 통해 서버의 엔드포인트(/emotionDiary)에서 데이터를 받아오고, 받아온 데이터 배열을 filter 함수로 걸러냅니다. 이때 조건은 로그인한 사용자의 name과 JSON Server 데이터의 name이 일치하는지 여부입니다.
  
  이 과정을 통해 로그인한 사용자와 동일한 이름을 가진 데이터만 추출할 수 있고, 최종적으로는 이 필터링된 데이터를 UI에 렌더링하여 화면에 표시합니다.
