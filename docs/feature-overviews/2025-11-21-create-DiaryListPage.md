# DiaryListPage

### 페이지 목적
- 여러 일기를 모아 정렬·검색·필터링을 통해 원하는 기록을 빠르게 찾기.
- 로그인 기반으로 JSON 서버에서 내 이름과 매칭되는 일기만 보여주기.
### 핵심 목표
- 데이터 출처(JSON 서버 vs 로컬스토리지) 구분 및 병행 관리.
- 감정/날짜/검색어 기반 탐색 기능 제공.
- 수정·삭제·작성 흐름을 목록에서 자연스럽게 이어가기.
### 구현 기능
- 데이터 관리
  - 9월·10월: JSON 서버에서 가져오기
  - 11월: 로컬스토리지에서 가져오기
  - 새 일기 작성 시 로컬스토리지에 저장w
  - 수정/삭제는 두 데이터셋 모두 가능
- 정렬/필터/검색
  - 최신순 / 오래된순 정렬 옵션
  - 감정 선택 필터링 (특정 감정만 보기)
  - 제목 검색 (검색어 입력 후 엔터로 실행)
- UI/UX
  - 감정별 지정색으로 카드 시각화 → 감정 상태 직관적 표현
  - idx 번호 표시 → 몇 번째 카드인지 확인 가능
  - 삭제 시 컨펌 팝업으로 경고 제공
  - 수정 버튼 클릭 시 모달창 → 작성 UI 재사용
  - 페이지네이션으로 하단에 5개 아이템으로 나눠서 보여줌
- 로그인 연동
  - 구글 로그인으로 사용자 정보(name) 가져오기
  - JSON 서버 데이터와 비교해 내 일기만 필터링
  - 프로필 사진 클릭 시 사용자 정보 표시 + 로그아웃 버튼
  
  
 ### 설계 근거
- JSON 서버: 여러 사용자 데이터가 섞여 있으므로 로그인 기반 필터링 필요.
- 로컬스토리지: 빠른 저장/조회, 개인 브라우저 환경에서 즉시 반영.
- 정렬·필터·검색: 일기 탐색 효율성을 높이고, 감정 기반 기록 습관 강화.
- UI 개선: 감정별 색상, idx 번호, 팝업 경고로 직관성과 안정성 확보.
 ###  기대 효과
- 사용자별 맞춤형 일기 목록 제공 → 협업/확장성 고려.
- 정렬·필터·검색으로 원하는 기록을 빠르게 찾을 수 있음.
- 수정·삭제·작성 흐름이 자연스럽게 이어져 사용성 강화.
