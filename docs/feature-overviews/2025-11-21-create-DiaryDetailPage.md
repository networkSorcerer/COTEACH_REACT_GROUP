# DiaryDetailPage

## 페이지 목적

- 선택한 일기의 전체 내용을 확인한다.
- 날짜, 감정, 제목, 본문 등 핵심 정보를 한 화면에서 제공한다.

## 핵심 목표

- URL 파라미터의 id 기반으로 특정 일기를 정확히 조회한다.
- Flask(PythonAnywhere) 감정 분석 API와 연결한다.
- 모바일 중심의 가독성 높은 상세 레이아웃을 구성한다.

## 구현 기능

### 데이터 관리

- JSON 서버 + 로컬스토리지 혼합 데이터 구조에서 단일 일기 조회를 안정적으로 처리한다.
- id 값을 기준으로 해당 일기의 실제 저장 위치(JSON 서버 또는 로컬스토리지)를 판단하여 로드한다.
- 수정/삭제 시에도 저장소 타입에 맞게 반영하며, 삭제 후에는 목록 페이지로 이동한다.

### 감정 분석

- Firebase Functions 기반 감정 분석은 과금 구조 문제로 사용을 중단했다.
- Python Flask로 감정 분석 API를 구축하고 PythonAnywhere에 배포했다.
- 상세 페이지에서 "감정 분석" 버튼 클릭 시 Flask API를 호출하여 결과를 받아 표시한다.
- 분석 실패 시 에러 메시지와 재시도 UI를 제공한다.

### UI/UX

- 날짜, 감정, 작성자 정보 블록을 상단에 배치해 메타데이터를 명확히 전달한다.
- 본문은 줄간격과 패딩을 넉넉하게 적용해 읽기 편하게 구성한다.

## 설계 근거

- 상세 페이지는 정보 전달 중심이므로 복잡한 요소를 제외하고 본문·감정 정보를 강조하도록 설계했다.
- 감정 분석 기능은 일기 전체 맥락을 볼 수 있는 상세 페이지에서 실행하는 것이 가장 자연스럽다.
- 혼합 데이터 구조에서도 id 기반 단일 조회는 가장 안정적인 접근 방식이다.

## 기대 효과

- 사용자에게 특정 일기의 맥락을 깊게 전달할 수 있어 기록의 가치가 올라간다.
- 감정 분석과 결합해 개인 기록에 대한 해석 기능이 확장된다.
